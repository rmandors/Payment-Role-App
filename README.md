## PaymentRole

PaymentRole is a desktop application built with Java and JavaFX to **manage employee payroll**.  
The application loads employee data, performs salary calculations, supports sorting and filtering, and allows exporting the processed information.

### Project structure

All paths below are relative to the `PaymentRole` folder:

- **`src/`**: Application source code.
  - **`src/PaymentRole.java`**: Main class that starts the JavaFX application.
  - **`src/controller/`**: Controllers and UI-related logic.
    - `Controller.java`: Main view controller.
    - `CalcManager.java`: Salary calculation logic.
    - `SortManager.java`: Employee sorting logic.
    - `ExportManager.java`: Data export functionality.
    - `ConsoleManager.java`: Console output handling.
    - `AlertManager.java`: Alerts and dialog handling.
    - `GUIObservers.java`: Observers used to keep the UI in sync.
  - **`src/model/`**: Domain / model classes.
    - `Employee.java`: Represents an employee.
    - `Manager.java`: Represents a manager (extending or composing `Employee`).
    - `EmployeeList.java`: Collection and management of employees.
    - `EmployeeComparator.java`: Comparators used for ordering employees.
  - Optional **`.fxml` files** for the views, such as `PaymentRole.fxml`, located in the same package as `PaymentRole.java` or in the appropriate resources folder.
- **`bin/`**: Compiled `.class` files generated by `javac`.
- **`lib/`**: External libraries required to run the application.
  - JavaFX (`javafx.base.jar`, `javafx.controls.jar`, `javafx.fxml.jar`, `javafx.graphics.jar`, etc.)
  - JAXB (`jaxb-api.jar`, `jaxb-runtime.jar`, etc.)
  - JSON Simple (`json-simple-2.3.0.jar`)
- **`employeesData.csv` / `EmployeeData.csv`**: CSV files containing employee data.
- **`README.md`**: Project documentation.

---

### Prerequisites

- **Java JDK 17+** (tested with `javac 23.0.1`).
- **JavaFX SDK 23.0.2** (or a compatible version).
- Additional libraries (if not using the ones already included in `lib/`):
  - JAXB RI
  - json-simple

The repository includes all required libraries in the `lib/` folder, so no extra installation is strictly necessary if that directory is used.

### How to build and run

The following commands assume the working directory is:

```bash
/Users/raymondportilla/Desktop/College/4th_Semester/Apps Programming/Final Project/PaymentRole
```

#### 1. Compile

```bash
cd "/Users/raymondportilla/Desktop/College/4th_Semester/Apps Programming/Final Project/PaymentRole"

javac \
  -d bin \
  --module-path lib \
  --add-modules javafx.controls,javafx.fxml \
  -cp "lib/*" \
  $(find src -name "*.java")
```

- **`-d bin`**: writes compiled `.class` files to the `bin` directory.
- **`--module-path lib`** and **`--add-modules javafx.controls,javafx.fxml`**: enable JavaFX using the JARs in `lib`.
- **`-cp "lib/*"`**: adds all additional dependencies (JAXB, json-simple, etc.) to the classpath.
- **`$(find src -name "*.java")`**: compiles every `.java` file in the project.

#### 2. Run

From the same `PaymentRole` folder:

```bash
java \
  --module-path lib \
  --add-modules javafx.controls,javafx.fxml \
  -cp "bin:lib/*" \
  PaymentRole
```

`PaymentRole` is the main class (`public class PaymentRole extends Application`).  
If the class is later moved into a package (for example `app.PaymentRole`), the fully qualified name must be used instead.

### Additional notes

- `PaymentRole.fxml` must be accessible from `PaymentRole.java`. The class uses:
  ```java
  FXMLLoader.load(getClass().getResource("PaymentRole.fxml"));
  ```
  which requires `PaymentRole.fxml` to be in the same package as `PaymentRole.java` or correctly configured on the resources classpath.
- On macOS, if JavaFX windows do not appear or a runtime error occurs, verify that:
  - The correct JDK is being used.
  - The `--module-path` option points to the directory containing the JavaFX JARs.
- The same compile and run configuration can be replicated in an IDE (IntelliJ IDEA, VS Code, Eclipse) by setting:
  - The Java SDK.
  - The JavaFX module configuration with `module-path` and `add-modules`.
